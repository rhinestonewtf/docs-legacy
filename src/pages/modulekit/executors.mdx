# Executors

Executors are smart account modules that are called during the execution phase of a UserOperation. They extend the execution logic of the account and thus allow for a more diverse set of actions that the account can natively perform.

## Building an executor

In order to build a compliant executor you need to ensure that it:

-   Inherits from the `PluginBase` contract.
-   Implements the following functions:
    -   `name()`
    -   `version()`
    -   `metadataProvider()`
    -   `requiresRootAccess()`
    -   `rhinestonePluginId()`
    -   `grantsPluginAccess()`

The template for a compliant executor (without actual logic) looks like this:

```solidity
contract ExecutorTemplate is PluginBase {
    function name() external view override returns (string memory name) {
        return "ExecutorTemplate";
    }

    function version() external view override returns (string memory version) {
        return "0.0.1";
    }

    function metadataProvider()
        external
        view
        override
        returns (uint256 providerType, bytes memory location)
    {
        return (0, "");
    }

    function requiresRootAccess() external view override returns (bool requiresRootAccess) {
        return false;
    }

    function rhinestonePluginId() external pure virtual override returns (bytes32) {
        return keccak256("ExecutorTemplate");
    }

    function grantsPluginAccess() external view virtual override returns (bool) {
        return false;
    }
}
```
